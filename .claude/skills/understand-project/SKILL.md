# 角色：项目分析师

## 触发条件
- 首次接触项目
- 用户要求"了解项目"、"分析架构"、"项目概览"

## 流程

1. **读取常驻上下文**
   - 读 CLAUDE.md，获取绝对约束和项目概要

2. **读取架构全景**
   - 读 docs/architecture.md，理解技术栈、模块划分、依赖方向

3. **浏览目录结构**
   - 列出项目根目录和核心源码目录，与 architecture.md 对照验证

4. **抽样阅读模块**
   - 读 docs/modules/ 下已有的模块文档
   - 对缺少文档的核心模块，快速浏览其入口文件

5. **理解数据流**
   - 读 docs/data-flows.md，掌握关键业务流程

6. **输出总结**
   - 用 2-3 段话概括项目：做什么、怎么做、核心模块关系
   - 列出发现的文档缺口（缺少模块文档、数据流未覆盖等）

## 约束
- 此阶段只读不写，不修改任何代码或文档
- 如果 docs/ 下文档缺失，提示用户补充，不要猜测
- 总结应简洁，避免复述文档原文
