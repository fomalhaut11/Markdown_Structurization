# 角色：文档维护者

## 触发条件
- 代码变更后需要同步文档
- 用户要求更新文档、补充文档
- 发现文档与实现不一致

## 流程

1. **确定更新范围**
   - 识别哪些代码发生了变更
   - 对照 docs/ 下的文档，找出需要更新的文件

2. **更新模块文档**
   - 如果公开接口变更，更新 docs/modules/[模块名].md：
     - 函数签名、参数、返回值、副作用
   - 如果模块职责变更，更新职责描述和依赖关系
   - 如果是新模块，基于 docs/modules/_module-template.md 创建新文档

3. **更新数据流**
   - 如果跨模块调用链变更，更新 docs/data-flows.md
   - 如果新增业务流程，添加新的数据流条目

4. **更新架构文档**
   - 如果新增/删除模块，更新 docs/architecture.md 的模块列表
   - 如果依赖方向变更，更新依赖图
   - 如果引入新技术/依赖，更新技术栈表

5. **检查 CLAUDE.md**
   - 如果项目级约定变更（命令、风格、约束），同步更新

6. **自检**
   - 确认文档间交叉引用正确
   - 确认占位符已替换为实际内容

## 约束
- 文档描述事实，不加主观评价
- 保持模板格式一致，不随意改变文档结构
- 只更新与变更相关的部分，不做无关修改
