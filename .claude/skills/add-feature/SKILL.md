# 角色：功能开发者

## 触发条件
- 用户要求添加新功能、新接口、新模块

## 流程

1. **理解需求**
   - 确认功能范围、输入输出、边界条件
   - 如果需求模糊，先向用户提问澄清

2. **定位影响范围**
   - 读 docs/architecture.md 确定涉及哪些模块
   - 读相关模块的 docs/modules/[模块名].md 了解现有接口
   - 读 docs/data-flows.md 确认是否影响现有数据流

3. **设计方案**
   - 说明将修改哪些文件、新增哪些文件
   - 说明新接口的签名、参数、返回值
   - 如果有多种方案，列出优劣让用户选择

4. **实现**
   - 遵循项目现有代码风格和约定
   - 新增公开接口必须包含类型定义
   - 编写或更新对应的测试

5. **验证**
   - 运行测试确保通过
   - 检查是否引入了不必要的依赖

6. **更新文档**
   - 如果新增/修改了公开接口，更新对应的 docs/modules/[模块名].md
   - 如果新增了跨模块数据流，更新 docs/data-flows.md

## 约束
- 不得修改不相关模块的代码
- 不得引入 architecture.md 未列出的新依赖（需要时先与用户确认）
- 每个公开接口必须记录副作用
